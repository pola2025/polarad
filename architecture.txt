[수정/보강] 폴라애드 웹사이트 기능 및 구조 기획서
1. 토스페이먼츠 결제 연동 기획
목표: 고객이 '올인원 패키지'에 대한 확신이 섰을 때, 상담 과정을 생략하고 즉시 결제하여 서비스를 빠르게 시작할 수 있도록 한다.
사용자 시나리오 (User Flow):
방문자가 [서비스 안내] 페이지에서 '올인원 패키지' 내용을 확인한다.
[상담 신청하기] 버튼과 함께 [330만 원(VAT 포함) 바로 결제하고 시작하기] 버튼이 나란히 노출된다.
[바로 결제하기] 버튼 클릭 시, 이름, 업체명, 연락처 등 간단한 정보 입력 폼이 나타난다.
정보 입력 후 '결제 진행' 버튼을 누르면 토스페이먼츠 결제창이 뜬다.
결제가 성공적으로 완료되면, "결제가 완료되었습니다. 담당자가 확인 후 24시간 내에 연락드리겠습니다." 라는 안내가 담긴 [결제 완료] 페이지로 이동한다.
동시에, 관리자(폴라애드)와 고객에게 결제 완료 안내 메일/알림이 자동으로 발송된다.
구현 방안:
프론트엔드 (Next.js): 결제 버튼 및 고객 정보 입력 폼을 만들고, 토스페이먼츠 결제창을 호출하는 스크립트를 연동한다.
백엔드 (Node.js): 토스페이먼츠 API와 연동하여 결제 요청을 생성하고, 결제 완료 후 그 결과를 검증하고 처리(DB에 저장, 알림 발송 등)하는 로직을 구현한다.
데이터베이스 (MySQL): payments 테이블을 생성하여 결제 번호, 고객 정보, 결제 금액, 결제 상태, 결제 일시 등을 기록한다.
필요 사항: 토스페이먼츠 판매자(가맹점) 가입 및 API 연동 키 발급.
2. 자동 견적서 발송 기능 기획
목표: 기본 패키지 외에 추가 페이지를 원하는 고객에게 즉각적인 맞춤 견적을 제공하고, 해당 고객의 정보를 확보하여 잠재 고객(리드)으로 전환한다.
사용자 시나리오 (User Flow):
방문자가 [서비스 안내] 페이지나 별도의 [맞춤 견적] 페이지로 진입한다.
'자동 견적 계산기'가 화면에 보인다.
기본 항목: 올인원 패키지 (300만 원) - 고정
선택 항목: "기본 10페이지 외에 추가 페이지가 필요하신가요?" → 추가 페이지 수 입력 (0~N)
사용자가 추가 페이지 수(예: 3페이지)를 입력하면, 실시간으로 총 견적 금액이 계산되어 표시된다.
예시: 3,000,000원(VAT별도) + (3 페이지 x 220,000원) = 총 3,660,000원 + VAT 10% = 4,026,000원(VAT포함)
계산된 금액 하단에 이름, 회사명, 이메일 주소를 입력하는 폼이 나타난다.
"입력하신 이메일로 상세 견적서 받아보기" 버튼을 클릭한다.
"견적서가 성공적으로 발송되었습니다." 라는 메시지가 표시된다.
사용자의 이메일함에는 즉시 깔끔하게 정리된 HTML 형식의 견적서가 도착한다. 동시에 관리자는 새로운 견적 요청 리드를 확보하게 된다.
구현 방안:
프론트엔드 (Next.js): 견적 계산 로직을 JavaScript로 구현하여 사용자가 입력하는 값에 따라 실시간으로 금액이 바뀌도록 만든다.
백엔드 (Node.js):
고객이 입력한 정보(이름, 이메일, 견적 내용)를 DB에 저장한다.
미리 만들어진 HTML 이메일 템플릿에 견적 내용을 채워 넣는다.
Nodemailer 같은 라이브러리와 SendGrid, AWS SES 등의 이메일 발송 서비스(SMTP)를 연동하여 고객에게 이메일을 발송한다.
데이터베이스 (MySQL): estimates 테이블을 생성하여 견적 요청 고객 정보, 견적 내용, 요청 일시 등을 기록하여 리드 관리에 활용한다.
