기획안 v1.0 | 2025-12-17

📊 현황 분석

보유 템플릿 (9종)

  • 1️⃣ 브랜드 소개 - 로고 + 헤드라인 + 피처
  • 2️⃣ 문제 제기 - 큰 텍스트 + 리스트
  • 3️⃣ 솔루션/패키지 - 3개 카드 레이아웃
  • 4️⃣ 자동화 시스템 - 아이콘 + 피처 뱃지
  • 5️⃣ Meta 리포트 - 대시보드 목업
  • 6️⃣ 프로모션 - 강조 텍스트 + CTA
  • 7️⃣ 홈페이지 제작 - 피처 리스트
  • 8️⃣ 인쇄물 - 아이템 리스트
  • 9️⃣ 영업 집중 - 간결한 메시지

현재 작업 프로세스

1 HTML 파일 열어서 수동 수정
2 브라우저에서 열기
3 스크린샷 캡쳐 도구 사용
4 캡션 수동 작성
5 파일 정리 및 저장

⏱️ 게시물 1개당 약 15-20분 소요

1. 입력

JSON 데이터로 콘텐츠 정의

2. 처리

EJS → HTML → Playwright 캡쳐

3. 출력

PNG 이미지 + 캡션 텍스트

⚙️ 기술 스택 제안

영역 선택 이유
런타임 Node.js 프로젝트 기존 환경, npm 생태계
템플릿 엔진 EJS HTML과 유사, 학습 비용 낮음
스크린샷 Playwright MCP 서버로 이미 설정됨, 안정적
CLI Commander.js 간단한 명령어 파싱
데이터 포맷 JSON 구조화, 에디터 지원 좋음

📋 구현 단계

1순위

단일 콘텐츠 생성 CLI

  • ✓ 기존 HTML → EJS 템플릿 변환 (9종)
  • ✓ JSON 데이터 스키마 정의
  • ✓ Playwright 스크린샷 기능
  • ✓ 캡션 자동 생성 (템플릿 기반)
  • ✓ 기본 CLI 인터페이스

사용법: npm run ig:generate -- --template intro --data content.json

2순위

일괄 생성 기능

  • ✓ 배치 JSON 파일 지원 (여러 게시물)
  • ✓ 순차/병렬 처리 옵션
  • ✓ 진행률 표시
  • ✓ 출력 디렉토리 자동 정리

사용법: npm run ig:batch -- --input batch.json --output ./output

3순위 (선택)

웹 UI

  • ○ 템플릿 미리보기
  • ○ 폼 기반 콘텐츠 입력
  • ○ 실시간 프리뷰
  • ○ 다운로드 버튼

별도 Next.js 페이지 또는 독립 앱으로 구현 가능

✨ 예상 결과

Before (수동)

  • ❌ 게시물 1개당 15-20분
  • ❌ 반복적인 수정 작업
  • ❌ 일관성 유지 어려움
  • ❌ 오타/실수 위험

After (자동화)

  • ✅ 게시물 1개당 1-2분 (데이터 입력만)
  • ✅ 명령어 한 줄로 생성
  • ✅ 템플릿 기반 일관성
  • ✅ JSON 검증으로 오류 방지
