## 1. 개요

### 1.1 문제 정의
Meta(Facebook/Instagram) 리드 광고에서 허위/장난 접수가 많아 광고주들이 고통받고 있음.

**현재 문제점:**
- Meta 인스턴트 폼의 원클릭 자동완성으로 진입 장벽이 너무 낮음
- 허위 전화번호, 허위 이메일로 장난 접수
- 욕설 등 악성 접수 발생
- 리드 품질 저하 → 영업팀 리소스 낭비
- 광고비 대비 실제 전환율 저조

### 1.2 솔루션
**카카오 로그인 기반 모바일 랜딩 페이지 서비스**

- 카카오 간편로그인으로 실명 인증된 사용자만 접수 가능
- 허위 정보 입력 원천 차단
- 욕설/장난 접수 심리적 억제 (본인 계정 연결)
- 진성 리드만 수집하여 광고 효율 극대화

### 1.3 서비스명
**폴라리드 (PolarLead)**

## 2. 서비스 구조

### 2.1 아키텍처
```
[광고주/고객]
     │
     │ 월 구독 결제
     ▼
[폴라리드 시스템 - 폴라애드 소유]
     ├── 랜딩 페이지 (Cloudflare Workers)
     ├── 카카오 로그인 앱 (폴라애드 명의)
     ├── 리드 데이터 저장소 (D1/KV)
     └── 관리자 시스템
     │
     │ 전용 URL 발급
     ▼
[광고주/고객]
     │
     │ Meta 광고에 URL 연결
     ▼
[최종 사용자 - 리드]
     │
     │ 카카오 로그인 → 정보 자동 수집
     ▼
[리드 데이터]
     │
     │ 알림 발송
     ▼
[광고주/고객]
```

### 2.2 고객 플로우
```
1. 폴라리드 서비스 신청 & 결제
2. 전용 랜딩 URL 발급받음
3. Meta 광고에 URL 연결
4. 리드 발생 시 알림 수신
5. 리드 데이터 확인 및 영업 진행
```

### 2.3 리드 수집 플로우
```
[Meta 광고 클릭]
       ↓
[모바일 랜딩 페이지] (360px 기준 반응형)
   - 프로모션 혜택 안내
   - 신뢰 요소 (성공사례 등)
       ↓
[카카오로 신청하기] 버튼
       ↓
[카카오 로그인] (검증용 - 완전 무료)
   - kakao_id 자동 수집 (핵심 - 법적 추적 가능)
   - 닉네임 자동 수집
   - 전화번호 자동 수집 (동의 시)
   - 이메일 자동 수집 (동의 시)
       ↓
[문의 내용 입력]
   - 욕설 필터 적용 (부적절한 표현 차단)
   - 차단 시도도 기록 (증빙용)
       ↓
[접수 완료]
   - Airtable에 저장 (IP, User-Agent 포함)
   - 텔레그램 실시간 알림 발송
   - 웹에서 완료 화면 표시
```

### 2.4 욕설 필터
- 클라이언트/서버 양쪽 검증
- 한국어 기본 욕설 리스트 적용
- 차단 시: "부적절한 표현이 포함되어 있습니다" 안내
- 차단 시도도 Airtable에 기록 (is_blocked: true)

## 4. 기술 스택

### 4.1 인프라
| 구성요소 | 기술 | 비고 |
|----------|------|------|
| 호스팅 | Cloudflare Workers | 서버리스, 글로벌 엣지 |
| 데이터베이스 | Airtable | 리드 데이터 저장, 증빙자료 |
| 구독 관리 | Cloudflare KV | 고객별 활성 상태 |
| 도메인 | 후이즈 DNS (CNAME) | 서브도메인 → Workers 연결 |
| 인증 | 카카오 로그인 API | 비즈니스 앱 (무료) |

### 4.2 Airtable 데이터 스키마
| 필드명 | 타입 | 설명 |
|--------|------|------|
| kakao_id | Text | 카카오 회원 고유번호 (법적 추적 가능) |
| nickname | Text | 카카오 닉네임 |
| phone | Phone | 전화번호 |
| email | Email | 이메일 |
| content | Long Text | 접수 내용 |
| ip_address | Text | 접속 IP |
| user_agent | Text | 기기/브라우저 정보 |
| is_blocked | Checkbox | 욕설 필터 차단 여부 |
| client_id | Text | 고객사 식별자 (서브도메인) |
| created_at | DateTime | 접수 시간 |

### 4.2 도메인 구조
```
*.polarlead.kr (또는 *.lead.polarad.co.kr)

예시:
- abc상사.polarlead.kr
- xyz병원.polarlead.kr
- 123학원.polarlead.kr
```

### 4.3 구독 상태 관리
```javascript
// Workers에서 요청 처리 시
1. 서브도메인에서 고객 ID 추출
2. D1/KV에서 구독 상태 조회
3. 활성 → 랜딩 페이지 렌더링
4. 만료 → "서비스 종료" 페이지 표시
```

## 5. 카카오 로그인

### 5.1 앱 구조
- **폴라애드 명의 비즈니스 앱 1개**로 전체 고객 서비스
- 고객별 앱 등록 불필요

### 5.2 수집 정보 (동의 항목)
| 항목 | 필수/선택 | 용도 |
|------|----------|------|
| 닉네임 | 필수 | 고객 식별 |
| 이메일 | 선택 | 연락처 |
| 전화번호 | 선택 (권장) | 영업 연락 |

### 5.3 비즈니스 앱 심사
- **심사 필요**: 전화번호 수집을 위해 필수
- **소요 기간**: 1~3일
- **필요 서류**:
  - 사업자등록증
  - 개인정보처리방침 URL
  - 서비스 설명

## 7. 리스크 및 고려사항

### 7.1 기술적 리스크
| 리스크 | 대응 방안 |
|--------|----------|
| 카카오 앱 심사 탈락 | 목적 명확히 작성, 개인정보처리방침 준비 |
| Workers 비용 증가 | 무료 티어 범위 내 운영, 캐싱 최적화 |
| 리드 데이터 유실 | D1 백업, 알림 즉시 발송 |

### 7.2 비즈니스 리스크
| 리스크 | 대응 방안 |
|--------|----------|
| 고객 이탈 | 연간 선결제로 락인 |
| 가격 저항 | ROI 명확히 제시 (허위 리드 비용 vs 구독료) |
| 경쟁 서비스 | 폴라애드 광고 대행과 번들링 |
