폴라애드(PolaAd) SEO 최적화 전략
AI 크롤러가 정보를 완벽하게 수집할 수 있도록 최적화

작성일: 2025년 11월 2일
버전: 1.0
기준: Google Search 2025, Next.js 15

======================================
🎯 SEO 전략 개요
======================================

[목표]
1. "온라인 영업", "홈페이지 제작", "B2B 마케팅" 등 핵심 키워드 상위 노출
2. 네이버 상호명 검색 광고 연동 최적화
3. 구글/네이버 검색엔진 크롤러의 완벽한 정보 수집
4. Core Web Vitals 점수 90점 이상 달성
5. Rich Snippets (풍부한 검색 결과) 표시

[핵심 전략]
✓ Next.js 15 Metadata API 활용 (최신 SEO 표준)
✓ Schema.org 구조화 데이터 (AI 크롤러 친화적)
✓ Core Web Vitals 최적화 (페이지 속도 + UX)
✓ Technical SEO (Sitemap, Robots.txt, Canonical)
✓ 컨텐츠 SEO (키워드 최적화, 내부 링크)

📊 현재 기획서 SEO 검토 결과

✅ 잘된 점 (Strengths)

1. Next.js 선택 ✓
   - SSR (Server-Side Rendering) 기본 지원
   - 검색엔진이 완전한 HTML을 받을 수 있음
   - SEO에 가장 유리한 프레임워크

2. 명확한 페이지 구조 ✓
   - 회사소개, 서비스 안내, 맞춤 견적, 포트폴리오, 문의하기
   - 계층적 구조로 크롤러가 이해하기 쉬움

3. 컨텐츠 카피 기획 ✓
   - 타겟 키워드 자연스럽게 포함
   - "온라인 영업", "24시간 영업사원" 등 명확한 가치 제안

⚠️ 보완 필요 사항 (Needs Improvement)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Metadata 정의 없음 ❌
   - 페이지별 Title, Description 미정의
   - Open Graph, Twitter Card 미기획

2. 구조화 데이터 없음 ❌
   - Schema.org LocalBusiness 마크업 필요
   - 서비스 정보, 가격 정보 구조화 필요

3. Technical SEO 미계획 ❌
   - Sitemap, Robots.txt 전략 없음
   - Canonical URL 규칙 없음
   - 이미지 최적화 가이드 없음

4. 성능 최적화 미계획 ❌
   - Core Web Vitals 목표치 없음
   - 이미지 lazy loading 전략 없음
   - 폰트 로딩 최적화 없음

5. 내부 링크 구조 미계획 ❌
   - 페이지 간 연결 구조 불명확
   - Breadcrumb 네비게이션 없음
