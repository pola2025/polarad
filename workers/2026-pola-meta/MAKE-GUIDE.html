Make + Airtable + Worker 연동 가이드

Airtable에 새 레코드가 추가되면 자동으로 알림 발송

진행 체크리스트

1

Make 시나리오 생성

  1. make.com 접속 → 로그인
  2. Create a new scenario 클릭
  3. 빈 시나리오 생성
2

Airtable 트리거 추가

  1. + 버튼 클릭 → Airtable 검색
  2. Watch Records 선택
  3. Connection 설정:
    • Create a connection 클릭
    • Airtable 계정 연결
  4. 설정값 입력:
3

HTTP 모듈 추가 (Worker 호출)

  1. Airtable 모듈 옆 + 클릭
  2. HTTP 검색 → Make a request 선택
  3. 설정값 입력:
Content-Type: application/json
⚠️ 아래 값에서 {{1.fields.Name}} 등은 Airtable 모듈에서 가져온 값입니다.
Make에서 직접 매핑하세요. 
{
  "name": "{{1.fields.Name}}",
  "phone": "{{1.fields.phone}}",
  "company": "{{1.fields.company}}",
  "industry": "{{1.fields.industry}}"
}
4

필드 매핑 방법

HTTP 모듈의 Body에서 각 값을 Airtable 필드와 연결:

  1. Body 입력창에서 "name": 뒤에 커서 위치
  2. 왼쪽 패널에서 Airtable - Watch Records 클릭
  3. fields → Name 선택
  4. 나머지 필드도 동일하게 매핑:
    • phone → fields.phone
    • company → fields.company
    • industry → fields.industry
5

테스트 및 활성화

  1. Run once 버튼 클릭
  2. Airtable에 테스트 레코드 추가
  3. Make에서 실행 결과 확인 (초록색 체크)
  4. 텔레그램/SMS/이메일 수신 확인
  5. 성공 시 Scheduling 켜기 (ON)
  6. 실행 간격 설정 (권장: 5분)
